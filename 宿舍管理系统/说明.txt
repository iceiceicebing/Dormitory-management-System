2. 总操作函数的解释

这几个大函数（如 StudentSystem、ManagerSystem 等）在系统架构中，扮演着“各组员独立工作站”的角色。
打个比方：
main.cpp 是这栋大楼的一楼安检大厅。安检大厅负责核对身份证和密码。一旦张三（学生）密码正确，main()作为登录端，就会把张三本人的档案袋（结构体指针StudentNode* student）交给他，并按一下电梯，把他送进专属的“学生业务办理大厅”——这，就是 StudentSystem 函数。

下面我以 StudentSystem 为例，给你深度剖析它的用法。其他组员拿到这个空函数后，应该按照这个“三步走”框架来写代码：
 🛠️ StudentSystem 内部的标准模板
其他组员在这个大函数里，其实就是要再写一个“子菜单循环”。

// student.cpp 里面
void StudentSystem(StudentNode* student) {
    int choice;
    
    // 1. 建立一个属于学生端的无限循环（直到他想注销退出）
    for (;;) {
        // 2. 打印学生专属的子菜单
        printf("\n===================================\n");
        printf("      欢迎同学：%s (学号:%s)\n", student->data.name, student->data.ID);
        printf("===================================\n");
        printf("      1. 查询水电费\n");
        printf("      2. 申请请假\n");
        printf("      3. 查看系统公告\n");
        printf("      0. 注销并返回主界面\n");
        printf("===================================\n");
        printf("请选择操作：");
        
        scanf("%d", &choice);
        
        // 3. 用 switch 处理具体业务
        switch (choice) {
            case 1:
                // 【用法示范：如何查水电费？】
                // 组员需要拿着当前学生的宿舍号，去遍历全局变量 all_dorms 找对应的房间
                // 比如：strcmp(all_dorms.data[i].room_number, student->data.room_number)
                printf("正在查询水电费...\n");
                // TODO: 写具体查询逻辑
                break;
                
            case 2:
                // 【用法示范：如何请假？】
                // 直接修改你传进来的 student 指针里的数据！极度方便！
                printf("请输入请假理由：");
                scanf("%s", student->data.on_leave.reason);
                student->data.on_leave.approved = -1; // -1代表待审批
                printf("请假申请已提交，等待导员审批！\n");
                break;
                
            case 3:
                // 【用法示范：如何看公告？】
                // 直接遍历全局变量 announcement_board
                printf("最新公告如下...\n");
                // TODO: 写遍历公告链表的逻辑
                break;
                
            case 0:
                // 注销登录，退回一楼大厅
                printf("注销成功，返回主菜单...\n");
                return; // 【极其关键】：这里用 return，直接结束这个大函数，控制权就自动交还给 main.cpp 了！
                
            default:
                printf("输入错误，请重新输入！\n");
                // 清理输入缓冲区，防止死循环
                while(getchar() != '\n');
                break;
        }
    }
}

💡 负责该端组员在使用时的“三大特权”
1. 拥有“上帝视角”的全局变量**
在这个大函数里，不需要传任何大表格进来。他们抬头就能看见 all_students、all_dorms、announcement_board（全局变量在函数中可以正常使用）。比如学生看公告，直接顺着 announcement_board 往下遍历打印就行了。
2. 拥有“VIP通行证”的参数指针**
参数 student 是极其宝贵的。组员不需要再去写什么“输入学号查找自己”，因为 student 指针就是当前登录的这个人。
 * 想知道他是谁？直接 student->data.name
 * 想改他的数据？直接 student->data.xxx = yyy。
   修改完后，不用管保存，只要 return 退出大函数，main.cpp 的 case 0 最后会负责把这些修改统统写进 txt 文件里。
3. 随时“安全撤退”的退路**
当学生办完事，输入 0 时，组员只需要写一句 return。
**千万叮嘱：绝对不要在这个大函数里写 exit(0);！** exit(0) 会直接把整个黑框框强行关掉，main.cpp 里的保存操作就触发不了了，数据会全部丢失。

 🔄 其他角色函数同理
 ManagerSystem(ManagerNode* manager)：宿管端。传进来的是宿管本人的指针。里面写宿管的专属菜单（1. 查房打分 2. 审批退宿 3. 发布公告...）。

3. 各个结构体存的内容解释

🧑‍🎓 1） 学生数据 (属于 all_students 链表)
大家拿到 student 指针后，通过 student->data.XXX 来访问或修改。
 基本信息：
   * name / ID：姓名 / 学号
   * school_name / class_name：学院 / 班级
   * dorm_building / room_number / bed_number：宿舍楼 / 房间号 / 床位号
   * contact_number / guardian_number：本人电话 / 监护人电话
 状态标志：
   * in_dorm：是否在寝（0--离开，1--在寝）
   * in_student_union：学生会身份（0--普通学生，1--干事，2--会长）
📝 办事申请：
   每个学生自带三个申请表：on_leave (请假)、dorm_alter (换宿)、dorm_exit (退宿)。
   这三个表内部结构一样，比如你想操作“请假”，就用 student->data.on_leave.XXX：
   * .reason：申请理由（**注意：录入理由时千万别敲空格，用下划线代替！**）
   * .start_date / .end_date：开始/结束日期（格式：20260427）
   * .approved：审批状态（-1--待审批，1--已批准，0--被驳回）
   * .text：导员/宿管的留言反馈

🏢 2）宿舍数据 (属于 all_dorms 顺序表)
⚠️ 避坑提醒：宿舍表是数组！访问时要用 all_dorms.data[i].XXX。
定位信息：dorm_building (楼栋)、room_number (房号)、school_name (所属学院)
生活缴费：
   * water_bill / elec_bill：水费余额 / 电费余额（浮点数）
舍员名单：
   * ID1 (寝室长)、ID2、ID3、ID4：存的是学号。如果是空床位，里面存的是 "无"。
💯 宿舍成绩：
   * 用 all_dorms.data[i].all_scores.scores[0] 到 [9] 存最近10次的分数。
   * 用 all_dorms.data[i].all_scores.count 看目前一共打了多少次分。
   * honor_info：获得的荣誉（比如“卫生标兵”）

🔑 3）宿管数据 (属于 all_managers 链表)
宿管端同学拿到 manager 指针后，用 manager->data.XXX 访问。
 * name / ID：姓名 / 工号
 * building_on_duty：负责值班的宿舍楼（比如 "文园1栋"）
 * duty_period：值班班次（0--早班，1--中班，2--晚班）

👨‍🏫 4）导员数据 (属于 all_teachers 链表)
导员端同学拿到 teacher 指针后，用 teacher->data.XXX 访问。
 * name / ID / school_name：姓名 / 工号 / 所属学院
🎓 管辖班级（关键！）：
   * 导员一个人可能管好几个班。管了几个班存在 classes.count 里。
   * 班级名称存在 classes.class_in_charge[0] 到 [9] 的数组里。遍历这个数组，再去学生表里找对应班级的学生即可。

 📢 5）公告数据 (属于 announcement_board 链表)
不管是谁，只要想看公告或者发公告，遍历这个链表就行。里面存的是 AnnounceInfo。
 * type：公告类型（0--重要通知，1--寻物启事，2--其他）
 * time：发布日期
 * heading：标题
 * sender：发布人
 * content：正文内容（支持包含空格的文本，具体长度可参考代码，如果不够可以直接改长度）

📋 6）查寝任务数据 (属于 all_inspections 链表)
学生会端的同学，你们的任务清单都在这里！里面存的是 InspectionInfo。
 * time：检查日期
 * dorm_building / room_number：要查哪个楼的哪个寝室
 * all_inspectors：谁去查的？名字存在 inspectors[0] 到 [9] 数组里，去了几个人存在 count 里。
 * score：打分结果
 * is_finished：任务状态（0--还没查，1--查完了并打完分了）

💡 终极开发法宝：
大家在自己模块里遇到需要**字符串对比**（比如判断寝室号是不是 "101"），或者字符串赋值（比如录入请假理由）时，千万记得：
 1. 对比必须用 if (strcmp(a, b) == 0)
 2. 赋值必须用 strcpy(a, b)
   （不要直接用 == 和 = 哦！）
